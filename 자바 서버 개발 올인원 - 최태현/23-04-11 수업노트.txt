## JDK, JRE, JVM
- JDK >JRE > JVM 순서로 포함하는 관계이다
- 우리가 작성한 코드를 컴퓨터가 이해하고 실행하게 하려면 컴퓨터가 이해할 수 있는 코드(0과 1로 이루어진 바이트 코드)로 바꿔주어야한다. 이 과정을 컴파일이라 하며 컴파일 해주는 프로그램을 컴파일러라고 부른다.
- 사용자가 작성한 코드는 OS별로 바이트 코드가 다르기에 같은 코드라도 다른 운영체제라면 정상적으로 작동을 하지 않을수도 있다. 그래서 OS별로 동일하게 동작할 수 있도록 JVM이 운영체제에 맞게 번역해서 전송해준다.
- JVM은 Java Virtual Machine의 약자이며 OS별로 존재한다. 바이너리 코드를 읽고 검증하고 실행한다.
- JRE(Java Rumtime Environment)는 자바 실행 환경의 약자이다.
- JRE 는 JVM + 자바 프로그램 실행에 필요한 라이브러리 파일등 확장개념이다.
- JDK(Java Development Kit)는 자바 개발 도구의 약자이다.
- JDK 는 JRE + 개발을 위한 도구(컴파일러, 디버그 도구)등이 포함되어 있다.

## 빌드와 실행
- 빌드란 소스코드파일을 컴퓨터에서 실행할 수 있는 독립 SW 가공물로 변환시키는 과정
- 빌드 세분화
1. 소스 코드를 컴파일 한다
2. 테스트 코드를 컴파일 한다
3. 테스트 코드를 실행한다
4. 테스트 코드 리포트를 작성한다
5. 기타 추가 설정한 작업들을 진행한다
6. 패키징을 수행한다
7. 최종 SW 결과물(Artifact)을 만들어 낸다
- 실행은 내가 작성한 코드(혹은 테스트 코드)를 컴파일을 거쳐 작동시켜 보는 것.
- 실행에서 독립SW 가공물이 나올 수도 있고, 나오지 않을 수도 있다.

## 빌드 툴
- 빌드를 수동적으로 했을때는 실수가 무조건 나오고 Resourse가 많이 드는 단점이 있다.
- 빌드 툴이란 소스코드의 빌드 과정을 자동으로 처리 해주고, 외부 소스 코드를 알아서 관리해주는 프로그램이다.
- Ant (예전 툴이고 현재는 사용되지 않음.)
1. 설정을 위해 xml을 사용한다.
2. 간단하고 사용하기 쉽다고 한다.
3. 복잡한 처리를 하려 하면 빌드 스크립트가 장황해져 관리가 어렵다
4. 외부 라이브러리를 관리하는 구조가 없다
- Maven
1. 설정을 위해 xml을 사용한다.
2. 외부 라이브러리를 관리할 수 있다.
3. 장황한 빌드 스크립트 문제를 해결했다.
4. 특정 경우에 xml이 복잡해진다.
5. xml 자체의 한계가 있다.
- Gradle
1. 설정을 위해 groovy 언어를 사용한다
2. 외부 라이브러리를 관리할 수 있다.
3. 유연하게 빌드 스크립트를 작성할 수 있다.
4. 성능이 뛰어나다.
