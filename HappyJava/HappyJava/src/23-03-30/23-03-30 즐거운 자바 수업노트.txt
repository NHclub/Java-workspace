## 상속
- 상속은 일반화 + 확장의 개념이다.
- TV라는 제품을 전자제품이라고 일반화할 수 있다. 이때 전자제품이 부모 TV는 자식클래스가 된다
- 상속은 굉장히 강한 결합이기에 웬만하면 사용 안하는게 좋다. 좋은객체가 되려면 응집도는 높이고 결합도는 약해야 한다.
//상속 선언 방법
[접근제한자] [abstract | final] class 클래스명 extends 부모 클래스명{
...
}
- 아무것도 상속받지 않으면 자동으로 java.lang.Object를 상속 받는다. 모든 클래스는 Object의 자손이다.
- 부모 타입으로 자손 타입을 참조할 수 있다.
Car car = new Bus(); // 자손을 참조하는데 부모의 기능만 사용 가능.

## 다형성 - 메소드 오버라이딩(Overriding)
- 상위 클래스의 메소드를 하위 클래스가 재정의 하는 것이다.
- 메소드의 이름은 물론 파라미터의 갯수나 타입도 동일해야 하며, 주로 상위 클래스의 동작을 상속받은 하위 클래스에서 변경하기 위해 사용된다.
- 메소드가 오버라이딩 되면 무조건 오버라이딩 된 메소드가 실행된다.
- 만약 자식의 메소드를 사용하려면 이렇게 할 수 있다.
Car car = new Bus(); // Bus 인스턴스 생성됨
Bus car2 = (Bus)car; // Bus클래스로 바꾼car를 Bus타입의 car2에 할당되니 메소드 사용 가능. Bus의 인스턴스는 이미 생성되었기 때문에 car를 Bus로 변경이 가능.
- 필드는 오버라이딩이 되어도 타입에 따라 타입필드값이 출력된다.
- 필드는 메소드를 통해서 사용할 수 있도록 코드를 작성해야함

## getter, setter
- 클래스의 필드값이 외부에서 변경되면 데이터의 무결성이 깨질 수 있기 때문에 필드는 pravite하게 선언하는것이 좋다.
- private한 필드를 접근하기 위한 메소드가 getter, setter이다.
- 메소드를 작성할 때 지역 변수인지 클래스의 필드인지를 구분짓기위해 this를 앞에 붙여 이것은 필드명이란것을 명시해야한다.
- setter, getter 메소드를 프로퍼티(property) 라고 말함.
- 필드와 프로퍼티의 차이점은 필드는 클래스 내부에 선언된 변수이며 무결성이 유지되어야 하므로 외부에서 쉽게 변경이 되면 안됩니다. 그래서 외부에서 접근하기 위해 만들어진 getter, setter 메소드를 프로퍼티라 부르며 필드는 클래스 내부에서만 사용되는 변수이고 프로퍼티는 외부에서 필드값을 가져오고 설정하는데 사용되는 메소드 입니다.
